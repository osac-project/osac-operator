## Project Instructions

### Validation Commands

Before your session ends, run these commands in order and fix all
failures. Lint failures block the PR — this is non-negotiable.

```bash
# 1. Regenerate if you touched CRD types (api/v1alpha1/*_types.go)
make manifests generate

# 2. Format, lint, and test
make fmt
make lint
make test
```

If you modified `go.mod`, also run `go mod tidy`.

If you modified `buf.gen.yaml`, also run `buf generate`.

### Coding Standards

- **Commit message format**: `OSAC-XXXXX: description of change`
  (Jira key from the ticket, lowercase description).
- **Dual-controller pattern**: Each resource has a resource controller
  (provisioning, lifecycle, finalizers) and a feedback controller (syncs
  state to fulfillment-service via gRPC). When fixing a bug, determine
  which controller is affected — changes to provisioning logic go in
  `{resource}_controller.go`, changes to state-sync logic go in
  `{resource}_feedback_controller.go`.
- **Reconciliation pattern**: Always save `oldStatus := obj.Status.DeepCopy()`
  before reconciliation. Compare with `equality.Semantic.DeepEqual` after.
  Only call `r.Status().Update()` if the status actually changed — otherwise
  you create infinite reconciliation loops.
- **Finalizer convention**: Resource controllers use
  `osac.openshift.io/{resource}-finalizer`. Feedback controllers use
  `osac.openshift.io/{resource}-feedback-finalizer`. Never remove a
  finalizer before cleanup fully completes.
- **Error handling**: Use `client.IgnoreNotFound(err)` for `Get()` calls
  in reconcile — the resource may be deleted between list and get. For
  parent lookups (e.g., Subnet -> VirtualNetwork), requeue with delay
  rather than setting Phase to Failed.
- **Test framework**: Ginkgo v2 + Gomega with `envtest` (real etcd +
  kube-apiserver). Tests live alongside controllers in
  `internal/controller/`. The test suite in `suite_test.go` bootstraps
  envtest with CRDs from `config/crd/bases` and `config/crd/fakes`.
- **Kubebuilder markers**: Use `+kubebuilder:validation:Required`,
  `+kubebuilder:validation:Pattern`, `+kubebuilder:validation:Enum`,
  `+kubebuilder:printcolumn` in type definitions.
- **AAP integration**: Networking controllers use
  `provisioning.RunProvisioningLifecycle()` with `OnBeforeProvision`,
  `OnSuccess`, `OnFailed` callbacks. Template names follow
  `{prefix}-{action}-{kind}` (e.g., `osac-create-subnet`).
- **Linter config**: golangci-lint v2 with `.golangci.yml`. Key enabled
  linters: dupl, errcheck, ginkgolinter, goconst, gocyclo, govet,
  ineffassign, lll, misspell, prealloc, revive, staticcheck, unconvert,
  unused. errcheck is relaxed in test files.

### Files You Must Never Edit

These are generated — your edits will be overwritten or cause CI to fail:

- `config/crd/` — generated by `make manifests`
- `zz_generated.deepcopy.go` — generated by `make generate`
- `internal/api/` — generated by `buf generate`
- `go.sum` — managed by `go mod tidy`
